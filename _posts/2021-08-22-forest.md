As you can see from above, it automatically did the nmap scan for us, Rustscan has proven to be very useful for me recently but I wouldn't 100% rely on it.

The nmap command it ran was: `nmap -vvv -p 53,88,135,139,389,445,464,593,3268,3269,5985,47001,49665,49664,49666,49667,49671,49676,49677,49684,49706,49920 10.10.10.161`

The options I regularly use are:

| `Flag` | Purpose |
| :--- | :--- |
| `-p-` | A shortcut which tells nmap to scan all ports |
| `-vvv` | Gives very verbose output so I can see the results as they are found, and also includes some information not normally shown |
| `-sC` | Equivalent to `--script=default` and runs a collection of nmap enumeration scripts against the target |
| `-sV` | Does a service version scan |
| `-oA $name` | Saves all three formats \(standard, greppable, and XML\) of output with a filename of `$name` |

## Road2r00t

Now we have the user flag, it's time to get NT_AUTHORITY. Since its an Active Directory box, we will use bloodhound. I'm not going to show how to setup bloodhound here though so lets get right to it.

We are going to get SharpHound onto the box, we can setup a python http server on our machine to we can download it from the victims box.

![](../../assets/img/htb/Forest/InvokeSharpHound.png)

We use `-ExecutionPolicy Bypass` to bypass the script execution policy.

Then we run `Invoke-Bloodhound` and use the `CollectionMethod All`

![](../../assets/img/htb/Forest/bloodhound-collectall.png)

Now we have the .zip, we need to get it onto our local machine. We can use SMBserver from impacket to do this, and on the victim box we can use `net use`

![](../../assets/img/htb/Forest/smbserver.png)

Now we can copy the file over from the box to our machine by using copy `*.zip z:`

Then by going to Bloodhounds GUI, we can click Upload Data on the right hand side and put in our zip file we just downloaded. Once we have done that we can use the analysis option within the GUI on the right side, and use `Find Shortest Paths to Domain Admins`

![](../../assets/img/htb/Forest/bloodhound-find.png)

Now we can get the graph:
![](../../assets/img/htb/Forest/bloodhound-graph.png)

Instantly we see that we belong to the group `EXCHANGE WINDOWS PERMISSIONS`, doing some research, there is many ways to attack this. A good article on exploiting this can be found here: <https://dirkjanm.io/abusing-exchange-one-api-call-away-from-domain-admin>. There is another way we could exploit this too, we can use `WriteDacl` to add us as a member of the group `Domain Admins`. Bloodhound shows us the steps we need to take in order to abuse this:
![](../../assets/img/htb/Forest/writedacl.png)
